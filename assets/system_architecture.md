## Sub-Skill Workflow Architecture

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      SUB-SKILL WORKFLOW ARCHITECTURE                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│   ┌──────────────────────────────────────────────────────────────────────┐│
│  │                      STEP 1: GATHER REQUIREMENTS                         ││
│  │  ┌─────────────────────────────────────────────────────────────────┐  ││
│  │  │ Input: User query, context, available materials                  │  ││
│  │  │ Process: Clarify object, scope, constraints, timeframe, language │  ││
│  │  │ Output: Structured requirements object                           │  ││
│  │  │ Gate: At least one object confirmed                             │  ││
│  │  └─────────────────────────────────────────────────────────────────┘  ││
│  └───────────────────────────────┬──────────────────────────────────────┘ │
│                                  │                                          │
│                                  ▼                                          │
│   ┌──────────────────────────────────────────────────────────────────────┐│
│  │                       STEP 2: COLLECT EVIDENCE                         ││
│  │  ┌─────────────────────────────────────────────────────────────────┐  ││
│  │  │ Input: Requirements object                                      │  ││
│  │  │ Tools: WebSearch, WebFetch, Read (knowledge base)               │  ││
│  │  │ Process: Fetch real-time data, authoritative docs, recent dev   │  ││
│  │  │ Output: Evidence bundle with sources + dates                     │  ││
│  │  │ Gate: Current data + 1 authoritative doc OR limitation flag     │  ││
│  │  └─────────────────────────────────────────────────────────────────┘  ││
│  └───────────────────────────────┬──────────────────────────────────────┘ │
│                                  │                                          │
│                                  ▼                                          │
│   ┌──────────────────────────────────────────────────────────────────────┐│
│  │                     STEP 3: CORE ANALYSIS                              ││
│  │  ┌─────────────────────────────────────────────────────────────────┐  ││
│  │  │ Input: Diver profile, dive objective, gas availability          │  ││
│  │  │ Tools: Read (knowledge base), Arithmetic                        │  ││
│  │  │ Process: Compute deco profile, gas selection, gas planning      │  ││
│  │  │ Output: Decompression profile, gas mix, risk assessment         │  ││
│  │  │ Gate: Bühlmann profile + GF stated, gas balanced, rule of 3rds │  ││
│  │  └─────────────────────────────────────────────────────────────────┘  ││
│  └───────────────────────────────┬──────────────────────────────────────┘ │

│                                  ▼                                          │
│   ┌──────────────────────────────────────────────────────────────────────┐│
│  │                    STEP 4: KNOWLEDGE UPDATE                            ││
│  │  ┌─────────────────────────────────────────────────────────────────┐  ││
│  │  │ Input: Analysis topics, keywords                                 │  ││
│  │  │ Tools: Read (knowledge base), WebSearch (gap fill, max 2)       │  ││
│  │  │ Process: Query SECOND-KNOWLEDGE-BRAIN.md, surface citations     │  ││
│  │  │ Output: 3-5 knowledge entries with Tier labels, flagged gaps    │  ││
│  │  │ Gate: ≥1 academic/authoritative source surfaced, coverage rated│  ││
│  │  └─────────────────────────────────────────────────────────────────┘  ││
│  └───────────────────────────────┬──────────────────────────────────────┘ │
│                                  │                                          │
│                                  ▼                                          │
│   ┌──────────────────────────────────────────────────────────────────────┐│
│  │                       STEP 5: ADVISOR                                  ││
│  │  ┌─────────────────────────────────────────────────────────────────┐  ││
│  │  │ Input: Core analysis + evidence + knowledge                     │  ││
│  │  │ Tools: Reasoning, Skill (sub-knowledge-updater optional)       │  ││

│  │  └─────────────────────────────────────────────────────────────────┘  ││
│  └───────────────────────────────┬──────────────────────────────────────┘ │
│                                  │                                          │
│                                  ▼                                          │
│   ┌──────────────────────────────────────────────────────────────────────┐│
│  │                    STEP 6: QUALITY GATE (MAIN)                        ││
│  │  ┌─────────────────────────────────────────────────────────────────┐  ││
│  │  │ Universal Gates (U1-U6) + Domain Gates (G1-G4)                 │  ││
│  │  │ Auto-fix + retry logic + degradation handling                   │  ││
│  │  │ Exit: All gates passed OR explicit limitation notice             │  ││

│  └───────────────────────────────┬──────────────────────────────────────┘ │
```
